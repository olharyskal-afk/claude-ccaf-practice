# CLAUDE.md

Этот файл содержит правила и соглашения для работы Claude Code в этом проекте.

## Project Context
Учебный репозиторий для подготовки к сертификации Claude Certified Architect – Foundations (CCA-F).

## Rules

### 1. Язык ответов
Всегда отвечай на украинском языке, если явно не указано иное.

### 2. Стиль кода
- Используй `const` и `let`, никогда `var`
- Все асинхронные операции через `async/await`, не callbacks
- Имена файлов в `kebab-case`
- Имена функций в `camelCase`

### 3. Обработка ошибок
- Всегда оборачивай async операции в `try/catch`
- Логируй ошибки с контекстом: `console.error('[module]:', error)`
- Никогда не игнорируй ошибки молча

### 4. Тестирование
- Писать тесты для каждой новой функции обязательно
- Тестовые файлы называть `*.test.js`
- Покрытие минимум 80%

### 5. Работа с файлами
- Не редактировать файлы в папке `/legacy`
- Не коммитить `.env` файлы и секреты
- Не коммитить без прохождения тестов

### 6. Git правила
- Commit message формат: `[тип]: краткое описание`
- Типы: `feat`, `fix`, `docs`, `refactor`, `test`
- Пример: `feat: додати обробку webhook подій`

### 7. MCP интеграция
- Все MCP инструменты должны иметь `description` с примером использования
- Ошибки возвращать через `isError: true` с категорией ошибки
- Максимум 5 инструментов на одного агента

## Never
- Не удалять продакшн данные без явного подтверждения
- Не использовать `console.log` в продакшн коде
- Не игнорировать TypeScript ошибки через `// @ts-ignore`
